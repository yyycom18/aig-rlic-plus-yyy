## [NEW COMPONENT – Indicator Evaluation Framework]
> The Indicator Evaluation Framework introduces three evaluation components—Indicator Identity Layer, Environment Interaction Radar, and Strategy Survival Radar—integrated at the top of each indicator's Apps overview page. These summarize identity, evidence, and strategy viability for a user-centric, narrative-first experience, without altering core dashboard workflows.

The App Dev agent acts as the orchestration layer for the Indicator Evaluation Framework.

This role includes assembling upstream artifacts from Research, Data, Econometrics, and Visualization agents and translating them into normalized evaluation summaries displayed in the dashboard.

The App Dev layer manages the integration of identity panels, environment interaction radar charts, and strategy survival radar charts into the indicator overview interface.

The evaluation layer also contains the score normalization and aggregation logic that maps econometric evidence and strategy metrics into radar-axis scores.

This mapping process transforms upstream analytical outputs into normalized evaluation scores used by the Environment Interaction Radar and Strategy Survival Radar.

The App Dev agent manages this orchestration logic while ensuring transparency between raw evidence artifacts and user-facing evaluation summaries.

**App Dev Agent Responsibilities**:
- Integrate `identity_panel.py`, `environment_radar.py`, and `strategy_radar.py` components into the Apps section of the indicator overview page, ensuring all three appear before existing Story/Evidence/Strategy/Methodology sections.
- Reference the indicator taxonomy (`data/indicator_taxonomy.json`), use case data, and computed result files (`results/environment_interaction_scores.json`, `results/strategy_survival_scores.json`).
- Display provenance, confidence, and source metadata in the appropriate evidence panels.
- All rendering should use the summary output files generated by the research/evidence/strategy pipelines, ensuring responsive, narrative-driven display with no change to underlying data, econometric, or visualization pipelines.

Outputs from Data, Research, Econometrics, and Viz agents feed these components. No modification of existing workflows is required or permitted.
